# NakaAnalytics Architecture

## System Overview

NakaAnalytics is an end-to-end ML-powered system that recommends optimal naka (checkpoint) locations, types, and time windows to maximize traffic violation detection.

## Components

### 1. Data Layer

- **Synthetic Data Generator**: Generates ~50,000 realistic traffic violation records with domain-specific patterns (DUI at nightlife zones, helmet violations near colleges, etc.)
- **SQLite Storage**: Stores violation events, recommendations, and model metadata

### 2. Pipeline Layer

- **Feature Engineering**: Creates cyclical encodings (hour, day-of-week), time buckets, grid cells, and categorical encodings
- **Training Pipeline**: End-to-end pipeline from data generation to model training
- **Inference Pipeline**: Loads models and generates predictions

### 3. Engine Layer

- **Clustering**: DBSCAN for hotspot discovery, KMeans for grid-based recommendations
- **Predictor**: XGBoost classifier predicting violation probabilities per location-time combination
- **Recommendation Engine**: Ranks locations by expected violation yield, assigns naka types
- **Drift Detector**: Monitors concept drift using PSI and KL divergence
- **Evaluation Metrics**: Precision@K, recall@K, uplift, hit rate

### 4. API Layer

- **FastAPI Server**: REST endpoints for recommendations, model status, clustering info
- **WebSocket**: Real-time updates to dashboard
- **Redis Streaming**: Message queue for violation event ingestion

### 5. Scheduler

- **APScheduler**: Runs retraining every 6 hours with drift detection

### 6. Dashboard

- **Frontend**: Real-time dashboard with Leaflet map, Chart.js charts, WebSocket live feed
- **Features**: Dark mode, glassmorphism styling, recommendation panel, live violation feed

## Data Flow

1. **Training Flow**:
   ```
   Synthetic Data вҶ’ Feature Engineering вҶ’ Clustering вҶ’ XGBoost Training вҶ’ Model Save
   ```

2. **Inference Flow**:
   ```
   Request вҶ’ Load Models вҶ’ Generate Candidates вҶ’ Predict Probabilities вҶ’ Rank & Recommend
   ```

3. **Real-time Flow**:
   ```
   Violation Event вҶ’ API Ingest вҶ’ SQLite Storage вҶ’ Redis Pub/Sub вҶ’ WebSocket вҶ’ Dashboard
   ```
